This article introduces the string matching problem, where the task is to locate all occurrences of a given pattern within a text. 
We begin with the naive string matching algorithm, the most straightforward approach that checks every possible alignment of the pattern.
Although inefficient for large inputs, it serves as the foundation for understanding more advanced algorithms studied later in this series.


# Introduction

The string matching problem is a fundamental task in computer science: given a text `T` of length `n` and a pattern `P` of length `m`, 
the goal is to find all positions in `T` where `P` occurs as a substring.

# Problem Structure

let `T[1:n]` is an array of `n` length, and `P[1:m]` is an array of `m` length.

<br/>

Example - 

```jsx
T = ['a', 'b', 'c', 'a', 'b', 'a', 'a', 'b', 'c', 'a', 'b', 'c', 'c']
 			 P =  ['a', 'b', 'a', 'a']
```

- `P` occur with shift `s` in text `T`


<br/>

i.e.

$$

0 < s < n-m

$$

and

$$
T[s+1: s+m] = P[1:m]

$$

<br/>


# Naive string-matching algorithm

In this algorithm, we align the pattern `P` with every possible position in the text `T` and check character by character if the pattern matches the substring of the text.

# Code


```cpp
#include <iostream>
#include <string>
using namespace std;

void naiveStringMatch(const string& T, const string& P) {
    int n = T.size();
    int m = P.size();

    for (int s = 0; s <= n - m; s++) {
        int j = 0;

        // Check if pattern matches at shift s
        while (j < m && T[s + j] == P[j]) {
            j++;
        }

        if (j == m) {
            cout << "Pattern occurs at shift " << s << endl;
        }
    }
}

int main() {
    string T = "ababaabab";
    string P = "abab";

    naiveStringMatch(T, P);

    return 0;
}

```

```markdown
Pattern occurs at shift 0
Pattern occurs at shift 5
```

# Time Complexity and Space Complexity  

- **Worst case:**  

For each shift, we may compare up to `m` characters, and there are \( n - m + 1 \) possible shifts.  

Example: \( T = a^n \) and \( P = a^m \).  

<div className='overflow-x-auto'>
$$
    Time Complexity = \mathcal{O}((n-m+1) \cdot m) \approx \mathcal{O}(nm) 
$$
</div>

- **Best case:**  

When all characters in the pattern `P` are distinct.  

Example: \( T = abcdeabcde \), \( P = bcd \).  

Since all characters are different:  

- for mismatch: \( s = s+1 \)  
- for match: \( s = s+m \)  

$$
    Time Complexity = \mathcal{O}(n)  
$$


- **Space Complexity:**  

The naive string matching algorithm only requires a constant amount of extra space for counters and indices.  

$$
    Space Complexity = \mathcal{O}(1)
$$
